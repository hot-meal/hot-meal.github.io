You May Not Need a Singe Page Application

The majority of websites don’t need the features that SPAs provide.

SPAs can make for great demos and lovely presentations, which leads to more support for the team, more funding, and this way, SPAs can improve the odds of success for a project.

However, SPAs then take away what they give. This is because they are quite error-prone and costly to develop.

There is so much state to track and so many built-in features that need to be reinvented.

A general rule of SPAs is that we always underestimate the complexity of handling state. Often by a large margin.

Also, we often underestimate the complexity of reimplementing history and link navigation. But we can get away with not caring because nobody really tests history management properly.

SPAs aren’t required for good UX design. Animation and fancy transitions aren’t good interactivity design and often can negatively impact accessibility, if not well implemented.
